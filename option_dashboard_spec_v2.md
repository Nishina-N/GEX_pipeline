# オプション需給・IV可視化ダッシュボード 実装仕様書 (Ver. 2.1)

## 1. コンセプトと基本運用
* **目的**: 2週間スイングトレードにおける「市場の壁」と「壁の脆弱性（IVの異常）」を毎朝1分で把握する静的地図。
* **更新頻度**: 1日1回（朝のルーティン時）。
* **現在価格 ($S_{current}$)**: 朝のデータ取得時点のSpot価格で完全固定。
* **配色**: 背景 `#FFFDD0` (Cream), 墨色 `#2C3E50`, 琥珀色 `#FFBF00`, 朱色 `#E74C3C`。

---

## 2. 時間軸（集計期間）の厳密な定義
チャート中央の2段折れ線を構成する「短期」と「長期」のGEX集計対象を以下と定義する。
* **左段（短期）**: 満期日（DTE: Days to Expiration）が**1週間以内（0〜7日）**にあるすべての限月（週次SQなどを含む）のGEXを合算。足元のヘッジの壁を示す。
* **右段（長期）**: **「次の月次SQ（メジャーSQ）」および「その次の月次SQ」**の合計2つの月次限月のGEXを合算。2週間スイングの本命となる岩盤を示す。

---

## 3. 計算ロジックと数式定義

### 3.0 モデル選択の方針

個別株オプションはアメリカン・オプション（早期行使可能）であるため、ヨーロピアン前提のBlack-Scholesモデルでは以下の場面でガンマを誤評価する：

* 配当落ち前日のディープITMコール（配当取り目的の早期行使）
* 急落後のディープITMプット

これらはWallの強度評価に直接影響するため、本仕様では**Barone-Adesi Whaley (BAW) 近似モデル**を採用し、早期行使プレミアムを考慮したガンマを算出する。

| 対象 | 採用モデル | 理由 |
|------|-----------|------|
| 個別株オプション | **BAW** | アメリカン・オプション。配当跨ぎ等で早期行使が発生するため |
| インデックスオプション（SPX等） | BS（参考実装） | 現金決済のため早期行使は事実上発生しない |

---

### 3.1 共通パラメータ定義

$$d_1 = \frac{\ln(S/K) + (r - q + \frac{\sigma^2}{2})T}{\sigma \sqrt{T}}, \quad d_2 = d_1 - \sigma\sqrt{T}$$

* $S$: 原資産価格 (Spot)
* $K$: 行使価格 (Strike)
* $T$: 満期までの期間（年率 = 残存日数 / 365）
* $\sigma$: インプライド・ボラティリティ (IV)
* $r$: リスクフリーレート
* $q$: 配当利回り（連続配当）
* $N(\cdot)$: 標準正規分布の累積分布関数
* $N'(x) = \frac{1}{\sqrt{2\pi}}e^{-x^2/2}$: 標準正規分布の確率密度関数

BSベースの補助変数：

$$M = \frac{2r}{\sigma^2}, \quad k = 1 - e^{-rT}, \quad h = \frac{2rT}{k}$$

---

### 3.2 BAWモデルによるオプション価格の算出

#### コール価格 $C_A$

早期行使臨界価格 $S^*$（以下の方程式をニュートン法で数値的に解く）：

$$S^* - K = C_{BS}(S^*) + \frac{S^*}{q_2}\left(1 - e^{-qT}N(d_1(S^*))\right)$$

$$q_2 = \frac{1 - M + \sqrt{(M-1)^2 + 4h/k}}{2}$$

$$A_2 = \frac{S^*}{q_2}\left(1 - e^{-qT}N(d_1(S^*))\right)$$

$$C_A(S) = \begin{cases} C_{BS}(S) + A_2\left(\dfrac{S}{S^*}\right)^{q_2} & S < S^* \\ S - K & S \geq S^* \end{cases}$$

#### プット価格 $P_A$

早期行使臨界価格 $S^{**}$（ニュートン法で求解）：

$$K - S^{**} = P_{BS}(S^{**}) - \frac{S^{**}}{q_1}\left(1 - e^{-qT}N(-d_1(S^{**}))\right)$$

$$q_1 = \frac{1 - M - \sqrt{(M-1)^2 + 4h/k}}{2}$$

$$A_1 = -\frac{S^{**}}{q_1}\left(1 - e^{-qT}N(-d_1(S^{**}))\right)$$

$$P_A(S) = \begin{cases} P_{BS}(S) + A_1\left(\dfrac{S}{S^{**}}\right)^{q_1} & S > S^{**} \\ K - S & S \leq S^{**} \end{cases}$$

---

### 3.3 BAWガンマの算出（数値微分）

BAWのガンマは解析解が複雑なため、**数値二次微分**で算出する：

$$\Gamma_{BAW}(S) = \frac{V_A(S + \Delta S) - 2V_A(S) + V_A(S - \Delta S)}{(\Delta S)^2}$$

* $V_A$: コールまたはプットのBAW価格
* $\Delta S = S \times 0.001$（標準的な刻み幅）

> **早期行使域での挙動**: $S \geq S^*$（コール）または $S \leq S^{**}$（プット）の場合、$V_A = S - K$ または $K - S$（内在価値のみ）となりガンマは実質ゼロ。BSではこの領域でガンマを過大評価するため、**ディープITMストライクのWall強度をBSは誇張する**。BAWではこれが正しく抑制される。

---

### 3.4 GEX (Gamma Exposure) の基本算出式

$$GEX_{call}(K) = +OI_{call}(K) \times \Gamma_{BAW,call}(S_{current}, K, T, \sigma) \times \text{Multiplier} \times S_{current}^2 \times 0.01$$

$$GEX_{put}(K) = -OI_{put}(K) \times \Gamma_{BAW,put}(S_{current}, K, T, \sigma) \times \text{Multiplier} \times S_{current}^2 \times 0.01$$

$$NETGEX(K) = GEX_{call}(K) + GEX_{put}(K)$$

> **設計思想**: コールはプラス（MMが原資産を買ってヘッジ）、プットはマイナス（MMが原資産を売ってヘッジ）として扱い、各ストライクの**正味のMM hedging pressure**を算出する。同一ストライクにコールとプット双方のOIが存在する場合、それらが互いに相殺された「実効的な壁の強度」がNETGEXに表れる。

---

### 3.5 IVの実値と予測値の算出（スキュー補正）

二重ヒストグラムを描画するため、2種類のIVを定義する。

**A. 実値ベース IV ($\sigma_{actual}$)**
* 前日終値時点で観測された各行使価格の最新IV。背面バー（太いバー）のGEX算出に使用する。

**B. 予測ベース IV ($\sigma_{pred}$)**
* 価格 $S$ が行使価格 $K$ に到達した（$S \to K$）と仮定した場合の予測IV。過去20日移動平均IV ($\sigma_{20MA}$) への回帰と、ボラティリティ・スキュー（$\frac{\partial \sigma}{\partial K}$）を考慮して算出する。

$$\sigma_{pred}(K) = \sigma_{20MA}(K) + \beta_{skew} \times (K - S_{current})$$

* $\sigma_{20MA}(K)$: 当該行使価格の過去20日間のIV移動平均。
* $\beta_{skew}$: スキュー勾配（近傍の行使価格間のIVの傾き $\frac{\sigma(K+\Delta) - \sigma(K-\Delta)}{2\Delta}$ などで近似）。

---

### 3.6 到達時の予測ガンマと予測GEX

前面バー（細いグレーのバー）を描画するため、価格が $K$ に到達した（$S = K$）状況をシミュレーションした予測GEXを算出する。

1. **予測ガンマ**: $\Gamma_{pred} = \Gamma_{BAW}(S=K, K, T_{new}, \sigma_{pred}(K))$
   * $S=K$ のためATMとなり、早期行使臨界価格との比較で $S < S^*$ が成立しやすい（= BSとBAWの差が最小化される点）。$T_{new}$ は到達までの想定日数を引いた残存期間。
2. **予測NETGEX**:

$$GEX_{pred}(K) = \left(OI_{call}(K) - OI_{put}(K)\right) \times \Gamma_{pred} \times \text{Multiplier} \times K^2 \times 0.01$$

## 4. 行使価格の選別ルール

### 4.1 基本方針：NETGEXによる統合評価

壁の強度は**NETGEX(K)**（コールとプットのガンマ露出を相殺した正味のMM hedging pressure）によって判定する。単純なOI量ではなく、「価格がそのストライクに近づいたときにMMが実際に行使するヘッジ圧力の大きさ」を基準とする。

> **OIベースとの違い**: OI量が多くても、ATMから遠いOTMストライクはガンマが小さいため、NETGEXも小さくなる。NETGEX順位付けにより、「見かけのOI壁」ではなく「実効的なガンマ壁」を抽出できる。

### 4.2 Call Resistance / Call Wall の抽出

1. 全ストライクについて $NETGEX(K)$ を算出する。
2. $NETGEX(K) > 0$ となるストライクを正値の降順（大きい順）に並べる。
3. **上位1本 = Call Resistance**（最強の上値抵抗。MMのショートヘッジ圧力が最大）。
4. **上位2〜3本 = Call Wall 2, Call Wall 3**（二次・三次の上値抵抗帯）。

```
Call Resistance = argmax_{K: NETGEX(K)>0} NETGEX(K)
Call Wall 2     = 2nd largest NETGEX(K) > 0
Call Wall 3     = 3rd largest NETGEX(K) > 0
```

**市場挙動の解釈**:
* 価格がCall Resistanceに接近すると、MMはデルタ中立維持のためショートを積み増す → 上値抑制。
* 投資家がコールを上のストライクへロールアップすると、Call Resistanceも上方シフトし価格が追随するケースがある（ガンマスクイーズの芽）。

### 4.3 Put Support / Put Wall の抽出

1. 全ストライクについて $NETGEX(K)$ を算出する。
2. $NETGEX(K) < 0$ となるストライクを負値の昇順（絶対値が大きい順）に並べる。
3. **上位1本 = Put Support**（最強の下値支持。MMのロングヘッジ圧力が最大）。
4. **上位2〜3本 = Put Wall 2, Put Wall 3**（二次・三次の下値支持帯）。

```
Put Support = argmin_{K: NETGEX(K)<0} NETGEX(K)
Put Wall 2  = 2nd most negative NETGEX(K) < 0
Put Wall 3  = 3rd most negative NETGEX(K) < 0
```

**市場挙動の解釈**:
* 価格がPut Supportに接近すると、MMはデルタ中立維持のためロングを積み増す → 下値支持（一時的なバウンス）。
* プットのITM化が進むと投資家がプットを行使・ロールダウンし、Put Supportが切り下がるケースもある（下落加速の転換点）。

### 4.4 HVL（High Volatility Level / Gamma Flip）の算出

**HVL**は $NETGEX(K)$ の符号が正から負へ（または負から正へ）切り替わる最近傍ストライクを指す。このレベルをまたぐと市場のボラティリティ特性が反転する。

1. 全ストライクを昇順に並べ、隣接ストライク間で $NETGEX$ の符号変化を検出する。
2. 符号変化が検出された区間において、線形補間でゼロクロス点を算出する。

$$HVL = K_i + (K_{i+1} - K_i) \times \frac{|NETGEX(K_i)|}{|NETGEX(K_i)| + |NETGEX(K_{i+1})|}$$

* $K_i$: 符号変化直前のストライク（$NETGEX > 0$ 側）
* $K_{i+1}$: 符号変化直後のストライク（$NETGEX < 0$ 側）

短期・長期それぞれ1本ずつ算出し、**現在価格に最も近いもの**を優先表示する。

**HVLの解釈**:
* 価格 > HVL: Positive GEX環境。MMのヘッジが価格安定方向に働くため、低ボラ・レンジ相場になりやすい。
* 価格 < HVL: Negative GEX環境。MMのヘッジが価格方向に追随するため、高ボラ・トレンド相場になりやすい。

### 4.5 Transition Zone（遷移帯）の定義

Put SupportとCall Resistanceの間の区間を **Transition Zone** と呼ぶ。この帯域内はNETGEXの正負が混在しており、価格が明確な方向性を持ちにくい中立ゾーンとなる。

* **上抜け確認**: 価格がCall Resistanceを上抜けかつ維持 → 短期強気バイアス。
* **下抜け確認**: 価格がPut Supportを下抜けかつ維持 → 短期弱気バイアス。
* **帯域内滞留**: レンジ戦略・様子見を優先。

### 4.6 重複・特殊ケースの処理

| ケース | 処理方針 |
|--------|---------|
| 同一ストライクに複数の役割が重複（例：Call Resistance かつ HVL） | 全ラベルを併記。表示上は役割ごとに色を分けて重ねて表示する |
| NETGEX > 0 のストライクが3本未満 | 存在する本数のみ表示。不足分は空白とする |
| NETGEX < 0 のストライクが3本未満 | 同上 |
| HVLが現在価格と一致またはゼロクロスが複数存在 | 現在価格に最も近いゼロクロス点を優先。次点を破線で補助表示 |
| 短期・長期で同一ストライクがWallに選出された場合 | 両方のラベルを同一行に併記し、GEXバーは各集計値を積み上げ表示 |

## 5. UI・可視化ルール

### 5.1 色の一意の定義（IV異常度の警告）
「最新IV」と「20日平均IV」の乖離率で色を決定する。
* **墨色 (Black)**: $\sigma_{actual} \le \sigma_{20MA} \times 1.2$ （通常状態）
* **琥珀色 (Amber)**: $\sigma_{20MA} \times 1.2 < \sigma_{actual} \le \sigma_{20MA} \times 1.5$ （警戒・イベント前）
* **朱色 (Red)**: $\sigma_{actual} > \sigma_{20MA} \times 1.5$ （異常・決壊リスク高）

### 5.2 右端：二重ヒストグラム
* **背面（太いバー）**: 実値 $NETGEX(K)$ を描画。色は上記5.1の判定に従う（墨/琥珀/朱）。
* **前面（細いバー）**: 予測 $GEX_{pred}(K)$ をグレーで重ねて描画。

### 5.3 中央：2段折れ線
* **線種**: 全て「実線 (Solid Line)」。
* **色**: 接続先のヒストグラムの色（墨/琥珀/朱）と連動。左右段でIV状態が異なる場合は中央で切り替える。斜線部分は右段（長期）の色を引き継ぐ。

### 5.4 注釈と限界点 (HVL)
* **ゼロガンマ注釈**: 端に `[+GEX → -GEX]` 等の反転方向を最小フォントで記載（色はライン色と連動）。
* **HVL (High Volatility Level)**: IVが「琥珀色」以上の行使価格帯でのみ出現する限界ライン。
